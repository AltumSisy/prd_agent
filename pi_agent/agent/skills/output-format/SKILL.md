# 输出格式指南 Skill

## 用途

规范 Agent 的输出格式，确保：
- 用户易于阅读和理解
- 关键信息突出显示
- 结构清晰美观

## 核心原则

**目标**：输出清晰、易读、结构化的回复

## 格式规范

### 1. 数据展示 - 使用卡片式布局，避免表格

❌ **不推荐**（表格在移动端和聊天界面显示不佳）：

```
| 字段 | 值 |
|---|---|
| VBELN | 0010000002 |
| MATNR | F010101MF01 |
| KWMENG | 90.000 |
```

✅ **推荐**（卡片式）：

```markdown
### 📋 订单详情

**销售凭证号**: `0010000002`  
**物料号**: `F010101MF01`  
**数量**: 90  
**金额**: ¥7,692.31

---

该订单包含 **1** 条行项目。
```

### 2. 关键值 - 使用行内代码高亮

✅ 示例：

```markdown
订单号 `0010000002` 的物料为 `F010101MF01`
```

### 3. 结构化列表 - 使用有序/无序列表

✅ 示例：

```markdown
1. **第一步**：检查程序 `ZORDER_CREATE`
2. **第二步**：验证数据是否存在
3. **第三步**：确认配置项
```

### 4. 状态标签 - 使用 emoji 辅助

| Emoji | 用途 |
|-------|------|
| ✅ | 成功 / 已确认 |
| ❌ | 失败 / 未找到 |
| ⚠️ | 警告 / 需要注意 |
| ℹ️ | 信息提示 |
| 📋 | 数据详情 |
| 🔍 | 分析结论 |
| 💡 | 建议 |

### 5. 代码/技术内容 - 使用代码块

✅ 示例：

```markdown
查询语句：
\`\`\`
SELECT * FROM VBAK WHERE VBELN = '0010000002'
\`\`\`
```

## 完整示例

### 查询结果展示

✅ **推荐格式**：

```markdown
### 📋 查询结果

**表名**: `VBAK`  
**条件**: `VBELN = '0010000002'`  
**结果**: ✅ 找到 1 条记录

---

**销售凭证号**: `0010000002`  
**销售组织**: `1000`  
**分销渠道**: `10`  
**客户**: `C001234`

---

该订单包含 **2** 条行项目，总金额 **¥15,384.62**
```

### 分析结论展示

✅ **推荐格式**：

```markdown
## 🔍 分析结论

经过代码分析和数据验证，问题原因如下：

### 1. 程序逻辑

程序 `ZORDER_CREATE` 在第 156 行检查物料类型时，
未处理 `MATNR` 为空的情况。

### 2. 数据验证

查询结果确认：
- 订单 `0010000002` ✅ 存在
- 物料号 `F010101MF01` ✅ 存在
- 物料类型 `MTART` ❌ 值为空

### 3. 修复建议

💡 建议采取以下措施：

1. 在程序中添加空值检查
2. 补充物料主数据中的物料类型
```

### 错误提示展示

✅ **推荐格式**：

```markdown
### ❌ 查询失败

**错误**: RFC 连接超时  
**表名**: `VBAK`  
**建议**: 请检查网络连接后重试

ℹ️ 如果问题持续，请联系系统管理员。
```

## 注意事项

1. **简洁性**：避免过度使用格式，保持内容简洁
2. **一致性**：同一类型的输出使用统一的格式
3. **移动端友好**：避免使用复杂的表格和长段落
4. **中英文兼容**：格式应该同时适用于中文和英文输出
